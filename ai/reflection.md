## 1. Which AI tools or models did I use?

## 2. Which prompt was the most useful? Why?

The most useful prompt throughout this process is the background setting for the agent, as it forms the underlying logic governing the agent. I can embed all my requirements within it, allowing the agent to carry out project development and implement various code functions in accordance with my instructions. I believe crafting effective background prompts for agents is extremely critical. If the specifications laid out in this background prompt are poorly structured, parts of the project will fail to align with your intended design. For instance, my first iteration of the project suffered cascading errors entirely due to an inadequately written agent background prompt; these mistakes rendered the project irreparable and forced a full rewrite. This clearly demonstrates just how vital well-crafted prompts are to the successful delivery and completion of any project.

## 3. Which AI-generated suggestion was wrong, incomplete, or misleading?

For example, the agent used hash codes while building a certain project. As a beginner working on Java projects, I couldn’t really grasp how hash codes are applied. What’s more, I’d already specified in the AI prompt that it shouldn’t employ knowledge beyond what I’ve learned. Even so, it still made that mistake. Fortunately, I spotted the issue while going through its code and fixed it. That’s one reason I think the AI doesn’t always generate accurate results.That’s why I always review all the code generated by AI to check whether it meets my requirements.

## 4. How did you check whether AI-generated code was correct?

First off, I read through the code to understand its underlying logic. After that, I create a new file in VS Code to rewrite and test that section of code myself. I’ll adopt the code only if it runs successfully. However, my technical ability is limited, so I ultimately send the code to my elder brother for a second review. I believe having experienced developers examine AI-generated code is an excellent practice. When your own knowledge isn’t sufficient to fully comprehend the code, seeking help from others is a wise choice. I never rely on other AI tools to audit the code, since all such outputs are machine-generated and prone to AI hallucinations that I cannot identify. My workflow goes like this: first I parse the code on my own, then run it locally, and finally consult seasoned developers. This process lets me verify whether the AI’s code output is correct.

## 11. Reflection on Recommendation Engine Addition

### What did this addition involve?

After the core project was complete, a Recommendation Engine was added. This feature recommends heroes and equipment based on five factors: hero type, player preference, win rate, equipment usage, and team composition.

### How was the AI used for this task?

I wrote a detailed 7-step prompt instructing the PM Agent to:

1. Analyse existing model classes (Hero, Equipment, Player, Team) to find missing fields
2. Design scoring formulas with weighted sub-scores and write them into docs/recommendation-formula.md
3. Implement RecommendationService with specific methods
4. Add recommendation menu options to Main.java
5. Supplement test data in DataInitializer
6. Update all documentation
7. Produce a final summary

The PM Agent first presented a design decision: whether to add a `heroWinRate` field to Hero.java (Option A, simple, 5 files to change) or calculate win rates from player data (Option B, complex). I chose Option A because it is beginner-friendly and straightforward.

### What did the AI generate?

- `RecommendationService.java` (~210 lines): weighted scoring engine with 13 methods
- `docs/recommendation-formula.md`: full formula documentation with sub-score tables and compatibility matrix
- Menu integration in `Main.java` with reason-based display (e.g. "team needs this role", "matches your preference")
- 3 new test cases (T16-T18) in test-cases.md

### What did I learn from this?

1. **Weighted scoring**: Learned how to combine multiple factors into a single score using decimal weights that sum to 1.0. This is a simple but useful algorithm idea that does not require machine learning.

2. **Design before coding**: The AI stopped to ask me to choose between Option A and Option B before writing code. This checkpoint approach prevented rushed decisions and ensured the implementation matched my skill level.

3. **Prompt structure matters**: Writing the 7-step prompt forced me to think through the entire feature before asking the AI to implement it. The clearer the requirements, the better the AI output.

### What remains for human review?

The recommendation test cases (T16/T17/T18) are marked "Pending manual test" because they require manual execution. I need to run the program, test the recommendation menu with different players, and verify the scoring makes sense before marking them as Pass.
